Functions

Definition: A binary relation R is a function if whenever (a, b) ∈ R and (a, c) ∈ R then we must have that b = c. In general, a relation of arity n is a function if whenever(a1, a2, . . . an 1, b) ∈ R and (a1, a2, . . . an 1, c) ∈ R then we must have that b = c. Notation: Since a function has this special property, we often use a special notation for being an element of a function. For example, if f is a binary relation that is a function, then instead of writing (a, b) ∈ f we usually write f (a) = b

Functions, Relations and Sets (class diagram)

A relation is a special kind of set. A function is a special kind of relation. Thus all of the relational operators(inverse, composition) can work with functions. ... and all of the set-theory operators(union, intersection etc.) can also workwith functions.
一个自变量只能对应一个应变量
比如说： is-a-parent-of = { (Henry VII, Henry VIII), (Henry VIII, Edward VI),(Henry VIII, Mary I), (Henry VIII, Elizabeth I) } 就不能看作一个函数，因为Henry VIII is a parent ofthree different people.

Defining functions: lambda notation

To define a function we can use enumeration (for finite functions) or comprehension, since every function is a set
For example, we defined the function S by comprehension as:
S=: {a ∈ Z, b ∈ Z | b = a + 1 • (a, b)}
We can define a function much more succinctly using the lambda notation:
T =: (λ a ∈ Z · a + 1)

Domain, codomain and image: definition

  • 定义域：The set X is called the domain of the function f
  • 陪域：The set Y is called the codomain (or range) of the function f
  • 值域：The subset of the codomain Y containing elements mapped-to by f is called the image of the function f 值域⊆陪域

Injective and Surjective Functions

  • injective (one-to-one) if each element in the image is mapped to by just one element in the domain; that is, if:(∀ x, y · (x ∈ X ∧ y ∈ X ∧ f (x) = f (y)) → x = y) Thus the inverse of an injective function is always a function (from theimage to the domain). 单射：定义域中所有的值都有值域中的值所对应，且一个定义域中的值只能对应一个值域中的值 值域中还有一些值没有定义域中的值所对应
  • surjective (onto) if each element in the codomain is mapped to by at least one element in the domain; that is, if:(∀ z · z ∈ Y → (∃ x · x ∈ X ∧ f (x) = z)) 满射：值域中的所有值都有一个定义域中的值与其对应 且某些值域中值有至少两个定义域中的值与其对应 For a surjective function the image is the whole codomain.
  • bijective if it is both injective and surjective. Thus if there is a bijection between two sets, then those sets have the same number of elements. 双射（也称一一对应）：值域中所有的值都有且只有一个定义域中的值与其对应

Injective: Hash Tables Example

Hash tables are a data structureused to map keys to values. Like an array, except the index(key) can be of any type. Sometimes called an associativearray (or, if the keys are strings,a dictionary). To implement this, you need ahash function that maps the keytype to an integer. Ideally, the hash function would be injective.

Total vs. Partial Functions

  • partial function(对函数进行限制，不是定义域中所有的值都取到) In certain cases we may want to allow some elements not to bemapped: this is known as a partial function. For example： reciprocal =: (λ a ∈ Z | a = 0 · 1/a) 应用：递归中的递归终止条件
  • total function(定义域中所有的值都可以取到) We can always make a partial function into a total function by restricting the domain to just those elements that are mapped from. (λ a ∈ (Z \ {0}) · 1/a)

Sets, bags, and sequences

A bag is like a set, but we also remember how many times an object occurs. A sequence is like a set, but we also remember where an object occurs.

Bags

A bag (or multiset) is like a set except that members are allowed to occur many times. The number of times an element occurs in a bag is called the multiplicity of that element. For example: [a, a, b, b, b, c] is the bag where a occurs twice, b occurs thrice and c occurs once. Notes: The notation [· · ·]is not very standard. The order of elements in a bag doesn’t matter.

A bag is really a special kind of function. For any set S, a “bag of S” is a function from S to N, giving the multiplicity of each element. Thus [a, a, b, b, b, c] is just a shorthand for {(a, 2),(b, 3),(c, 1)} For example, given some set F , {apple, orange, pear},given the bag myShopping , [apple, apple, pear, pear, pear] we can say myShopping = {(apple, 2),(orange, 0),(pear, 3)}

Operations on Bags

  • 对于 ∪ / ∩ 来说，只是一种关系，而不是函数，所以不能把它合成一个bag： [ a, a, b, b, b, c ]] ∪ [[ a, b, b, c, c ] = {(a, 2),(b, 3),(c, 1)} ∪ {(a, 1),(b, 2),(c, 2)} = {(a, 2),(b, 3),(c, 1),(a, 1),(b, 2),(c, 2)}
  • a special union operator [ a, a, b, b, b, c ]⊕ [ a, b, b, c, c ] = [ a, a, a, b, b, b, b, b, c, c, c ] Definition: for any two bags A and B A ⊕ B = (λ x · A(x) + B(x))

Sequence 记录元素所在的位置

A sequence is like a set except that we remember the order in which members occur The place where an element occurs in a sequence is called the position of that element. For example: < a, b, c, b, a, a > Notes: The notation <· · ·> is not very standard. We usually start counting position at 1 (unlike arrays)

A sequence is really a special kind of function For any set S, a sequence of S is a (partial) function from N to S,giving the element at each position. Thus < a, b, c, b, a > is just a shorthand for {(1, a),(2, b),(3, c),(4, b),(5, a)} For example, given some set F =: {apple, orange, pear}, given the sequence myEating =:< happle, pear, apple, pear, apple > we can say myEating = {(1, apple),(2, pear),(3, apple),(4, pear),(5, apple)}

  • 对于 ∪ / ∩ 来说，只是一种关系，而不是函数: Example: < a, a, b > ∪ < a, b > = {(1, a),(2, a),(3, b)} ∪ {(1, a),(2, b)} = {(1, a),(2, a),(2, b),(3, b)}
  • cardinality The cardinality of a finite set is the number of elements it contains. 用#S 来表示 Since a sequence is a set of tuples, the cardinality of a sequence is also its length. 如：#< a, b, c, b, a > = 5 The empty set ∅ is also a sequence; when we want to refer to it as a sequence we usually write < > #< > = 0
  • concatenation 就是按顺序排起来 <a, b, c>⌒<b,a> = <a,b,c,b,a> <a, b, c>⌒<b,a>⌒<b,a> = <a,b,c,b,a,b,a>
