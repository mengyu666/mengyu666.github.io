字符数据类型和操作

Unicode 和ASCII 编码

一.Unicode

1.Unicode标准定义了三种字符编码方式

UTF-8

  • 与ASCII编码表一样
  • 用8bits来表示一个字符，总共能表示256个字符
  • 是互联网标准中默认的编码形式，如超文本标记语言

UTF-16

  • 用16bits来表示一个字符，总共能表示65536个字符
  • 可以表示大多数的亚洲语言

UTF-32

  • 用32bits来表示一个字符

    2.Unicode代码点（码位）（Unicode Code Point）

    在Unicode代码标准中，每个字符被赋予一个值，这个值叫做代码点。
    代码点的表示方法：U+一个十六进制的值
    如 char cha =‘U+0041’；
    Unicode 代码点的范围[U+0000-U+10FFFF]
    Unicode标准最多用21bits来表示一个字符
    在Java中，一个代码点用32bits表示

  • 低21bits表示一个代码点

  • 高11置0

    3.Unicode 平面

    Unicode的编码范围为U+0000 ~ U+10FFFF，一共包含216∗17=1114112216∗17=1114112个码位。
    整个编码空间划分为17个平面（plane），每个平面包含216=65536216=65536个码位（codepoint）。
    其中0号空间叫做基本多文种平面（Basic Multilingual Plane, BMP），其他平面叫做补充平面（supplementary planes）。
    Java字符类型只能表示在BMP平面的字符

    4.Unicode 代码单元(Unicode Code Unit)

    代码单元是由UTF-X形式来定义的

  • 在UTF-8中，一个代码单元是8bits

  • 在Java中，字符类型用UTF-16来表示一个字符，因此一个char就表示一个代码单元。由用UTF-X的何种形式决定，一个Unicode 代码点能表示一到多个Unicode 代码单元。

    5.补充字符

    Java用16bits来表示一个字符类型，因此他只能表示BMP中的字符。
    一个16bits最多只能表示65536个字符，超过此范围的字符被称为补充字符，即1到16平面。

    字符型数据与数值型数据之间的转换

    char类型可转换为任意一种数据类型，而整数类型转换为char类型时，只能用该数据的低16位。

  • 要将一个浮点数转换为char类型，首先要将浮点数转换为int类型，再将这个整数类型转换为char类型。

  • 当一个char类型数据被转化为整数类型时，这个字符的Unicode就被转化为某一个指定的数据类型。

  • 如果转换结果适用于目标变量，就可以使用隐形转换方式；反之，要使用显性转换方式。

  • [0000-FFFF]之间任何一个十六进制正整数都可以隐式转换为char型数据。

  • 所有的数值操作符都可以用于char类型中：
    如果字符+字符（或数字） 结果是数字

如果字符+字符串 结果是字符与该字符串相连

字符的比较

两个字符可以用操作关系符进行比较。这是通过比较两个字符的Unicode值实现的。

|–|-|–|

|’A’-‘Z’|65-90|\u+0041-\u005A|
