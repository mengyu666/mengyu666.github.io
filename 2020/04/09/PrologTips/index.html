PrologTips

Author: master Date: April 9, 2020  16:58:23 Category: Logic

Consider using the underscore rather than a named variable. The underscore is just like a variable, except you are saying to Prolog that you don’t really care what value it gets during unification. Unlike variable names, underscores can have different values in the same unification.
For example: father(X,X) will not unify with father(vader,luke) because X cannot be 2 different values (vader and luke)
However: father(,) will unify with father(vader,luke) because Prolog considers both underscores to be different variables. The first one will unify with vader, the second with luke.
